# Benchmark Taxonomy Mapping — travel_a2a_v2 Formal Workload

```
experiment_version: travel_a2a_v2
environment_type:   a2a_inspired_travel
```

No external benchmark data was imported into this project. This document
maps the formal workload's task/attack categories to the GENERAL categories
those kinds of tasks/attacks are known by in the wider agent-benchmark and
prompt-injection literature — for framing this work relative to that
literature in the WISA 2026 poster, not as a claim of having reused any
specific published dataset.

**Every row below is labeled explicitly**: `benchmark-inspired category` (a
general category name that also appears in the literature) vs.
`independently authored fixture` (this project's own task/content/attack
design, not copied from or validated against any external corpus).

## Explicit non-claims

- This project did not clone, execute, or statistically validate against
  any of InjecAgent, AgentDojo, BIPIA, or any other published agent-security
  benchmark — the category names above are used only as shared vocabulary
  for describing what KIND of task/attack each workload element resembles.
- No task instance, content record, or attack payload in this workload was
  copied from an external source; all are generated by
  `formal_workload_generator.py` (tasks/content) or hand-authored JSON
  configs (`configs/travel_a2a/attacks/*.json`) specific to this project's
  5-agent architecture.
